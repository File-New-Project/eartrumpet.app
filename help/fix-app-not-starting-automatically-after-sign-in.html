Fix packaged applications not starting automatically after sign in

In Windows 10 and Windows 11, certain packaged applications (such as EarTrumpet) may not start automatically after sign-in if required startup policies are missing. These policies are often improperly removed by ineffective or poorly designed scripts and tools that claim to “debloat” the operating system, fix issues, or enhance privacy—actions that can cause more problems than they solve.

Symptoms

  • The application fails to start automatically after sign-in.
  • The application continuously appears as "Disabled" in Task Manager > Startup Apps.
  • The application cannot be configured to start automatically in Settings > Apps > Installed apps > [App Name] because the toggle is disabled.
  • The application cannot be configured to start automatically in Settings > Apps > Startup because the toggle is disabled.
  • The application reports that startup is disabled by Group Policy, even when no such policy exists.
  • The application's built-in setting to manage startup is disabled or grayed out.

Resolution (automatic)

  1. Download RestoreStartupTaskPolicies.reg.
  2. Right-click the downloaded file and select Merge.
  3. If prompted by User Account Control, click Yes to allow changes.
  4. When asked to confirm adding the information to the registry, click Yes.

Resolution (manual)

  1. Go to Start, type Notepad, and select it.
  2. Copy the following registry script: 
            [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System]
        "EnableFullTrustStartupTasks"=dword:00000002
        "EnableUwpStartupTasks"=dword:00000002
        "SupportFullTrustStartupTasks"=dword:00000001
        "SupportUwpStartupTasks"=dword:00000001
  3. Paste the script into Notepad. Select File > Save As, set Save as type to All Files, and name it RestoreStartupTaskPolicies.reg. Select Save.
  4. Right-click the saved file and select Merge. Confirm in the User Account Control dialog.
  5. When prompted, select Yes to confirm adding the keys to the registry, then select OK in the confirmation message.
